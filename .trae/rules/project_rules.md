在实现日志输出方法时，使用项目中logger命名空间下的方法输出日志，它包含Info、DebugLog、Warning、Error方法，用于输出不同级别的日志信息，使用时需要的参数：
Info(string module, string message, Object context = null)
其中：
module：日志所属的模块，使用LogModules脚本中定义的枚举值。
message：日志的具体信息，描述发生的事件或问题。
context：可选参数，用于提供额外的上下文信息，例如对象引用、变量值等。

LogModules脚本定义了各个模块的枚举值，用于在日志输出时指定日志所属的模块，在需要定义新的大类模块时，需要在LogModules脚本中添加新的枚举值并使用它。

本项目使用的是unity中的input system，所有的输入事件都通过该系统处理，在项目中由Assets\Scripts\Core\InputManager.cs统一管理,编写输入事件处理方法时，需要使用InputSystem类的方法来获取输入值而不是直接监听键位。

项目中多数视觉效果通过动画状态机实现，在编写脚本时不必通过脚本实现动画效果，而是通过动画状态机来控制。
游戏本体是基于tilemap的2D游戏，计算坐标时需要注意tilemap的坐标系统与世界坐标系统的差异。
目前设计上，游戏地图由多个tilemap组成，每个tilemap表示游戏中的一个层级，例如地面、墙壁、障碍物等。
而大部分可交互物体为单独的对象，不在tilemap中，例如单词、敌人等。

关于移动，目前的设计为4个方向移动，分别为上、下、左、右。
移动时，玩家角色会向目标方向移动一个单位长度，单位长度为1个tilemap单元格的大小。
其它组件也是类似逻辑，如果有特殊行为会额外说明并在代码中注释。

除了unity自带的layer，项目中目前自定义的layer有：
- Obstacle：障碍物层，用于表示游戏中的障碍物，例如墙壁、敌人等。
- Player：玩家层，用于表示游戏中的玩家角色。
- Wall：墙壁层，用于表示游戏中的墙壁。
- Ground：地面层，用于表示游戏中的地面。
- Interactable：可交互层，用于表示游戏中的可交互对象。
- Background：背景层，用于表示游戏中的背景。

自定义的tag有：
- WalkableArea：可走层，用于表示游戏中的可走区域。
- Word：单词层，用于表示游戏中的单词对象。
- Obstacle：障碍物层，用于表示游戏中的障碍物。

UI实现使用的是MVC架构，且均继承于Assets\Scripts\UI\Core\BaseMVC中的三个基类：
- BaseView：视图基类，用于表示UI中的视图层，负责显示界面元素和接收用户输入。
- BaseController：控制器基类，用于表示UI中的控制器层，负责处理用户输入和更新模型数据。
- BaseModel：模型基类，用于表示UI中的模型层，负责存储和管理界面元素的数据。
需要定义新的UI模块时，需要继承以上三个基类，并在Assets\Scripts\UI目录下创建新的文件夹，用于存放新模块的脚本文件。

对象行为控件使用的是行为树架构，每个对象都有一个行为树，用于表示对象的行为逻辑。
现在的实现逻辑是编写最小动作组件，由最小动作组件组合成复杂行为，复杂行为组成行为树节点。

实现单例模式时，一般通过继承项目中的Assets\Scripts\Core\Singleton\Singleton.cs脚本，在需要定义新的单例类时，需要继承此基类并指定泛型参数为新的单例类。
