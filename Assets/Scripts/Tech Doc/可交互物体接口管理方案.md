# 可交互物体接口管理方案

## 1. 概述

本技术方案详细描述如何在Unity中实现场景中可交互物体的接口管理系统。该系统将提供统一的交互接口、管理机制和事件处理，使不同类型的可交互物体能够以一致的方式与玩家和其他游戏系统交互。

## 2. 核心设计原则

1. **接口统一性**：所有可交互物体实现统一的接口
2. **可扩展性**：支持多种类型的交互行为和效果
3. **模块化**：将交互逻辑与物体本身解耦
4. **事件驱动**：使用事件系统处理交互触发和响应
5. **可配置性**：允许通过编辑器配置交互属性

## 4. 交互管理器实现

### 4.1 交互管理器核心类

```csharp
/// <summary>
/// 交互管理器，负责管理场景中的所有可交互物体
/// </summary>
public class InteractionManager : Singleton<InteractionManager>
{
    #region 字段
    
    // 当前可以交互的物体
    private List<IInteractable> currentInteractables = new List<IInteractable>();
    // 交互距离
    [SerializeField] private float interactDistance = 2f;
    // 交互提示UI
    [SerializeField] private InteractPromptUI promptUI;
    // 玩家Transform
    private Transform playerTransform;
    
    #endregion
    
    #region 生命周期
    
    private void Start()
    {
        // 获取玩家Transform
        playerTransform = GameObject.FindWithTag("Player").transform;
    }
    
    private void Update()
    {
        // 检测周围可交互物体
        DetectInteractables();
        // 处理交互输入
        HandleInteractInput();
    }
    
    #endregion
    
    #region 交互检测与处理
    
    /// <summary>
    /// 检测玩家周围的可交互物体
    /// </summary>
    private void DetectInteractables()
    {
        currentInteractables.Clear();
        
        // 使用OverlapCircle检测周围物体
        Collider2D[] colliders = Physics2D.OverlapCircleAll(
            playerTransform.position, 
            interactDistance
        );
        
        foreach (Collider2D collider in colliders)
        {
            // 检查物体是否带有可交互组件
            IInteractable interactable = collider.GetComponent<IInteractable>();
            
            if (interactable != null && interactable.CanInteract(playerTransform.gameObject))
            {
                currentInteractables.Add(interactable);
            }
        }
        
        // 按优先级排序
        currentInteractables.Sort((a, b) => b.Priority.CompareTo(a.Priority));
        
        // 更新交互提示UI
        UpdatePromptUI();
    }
    
    /// <summary>
    /// 处理交互输入
    /// </summary>
    private void HandleInteractInput()
    {
        // 检查是否有交互输入
        if (InputManager.Instance.InputActions.GamePlay.Interact.triggered && currentInteractables.Count > 0)
        {
            // 与优先级最高的物体交互
            currentInteractables[0].Interact(playerTransform.gameObject);
        }
    }
    
    /// <summary>
    /// 更新交互提示UI
    /// </summary>
    private void UpdatePromptUI()
    {
        if (currentInteractables.Count > 0)
        {
            // 显示优先级最高的物体的交互提示
            string prompt = currentInteractables[0].GetInteractPrompt();
            promptUI.Show(prompt);
        }
        else
        {
            // 隐藏交互提示
            promptUI.Hide();
        }
    }
    
    #endregion
}
```

## 5. 可交互物体基类实现

### 5.1 基础可交互组件

```csharp
/// <summary>
/// 可交互物体的基础实现
/// </summary>
public abstract class InteractableBase : MonoBehaviour, IInteractable
{
    [Header("交互设置")]
    [SerializeField] protected InteractionType interactionType = InteractionType.Examine;
    [SerializeField] protected int priority = 1;
    [SerializeField] protected string interactPrompt = "按 E 键交互";
    [SerializeField] protected bool isInteractable = true;
    
    // 物体类型
    protected ObjectType objectType = ObjectType.None;
    
    #region IInteractable 接口实现
    
    public InteractionType Type => interactionType;
    
    public int Priority => priority;
    
    public virtual bool Interact(GameObject interactor)
    {
        if (!CanInteract(interactor))
        {
            return false;
        }
        
        // 执行具体交互逻辑（由子类实现）
        return PerformInteraction(interactor);
    }
    
    public virtual bool CanInteract(GameObject interactor)
    {
        return isInteractable;
    }
    
    public virtual string GetInteractPrompt()
    {
        return interactPrompt;
    }
    
    #endregion
    
    /// <summary>
    /// 执行具体的交互逻辑
    /// </summary>
    /// <param name="interactor">交互发起者</param>
    /// <returns>交互是否成功</returns>
    protected abstract bool PerformInteraction(GameObject interactor);
}
```

/// <summary>
/// 可使用的机关实现
/// </summary>
public class UsableMechanism : InteractableBase, IPropBehavior
{
    [Header("机关设置")]
    [SerializeField] private bool isActivated = false;
    [SerializeField] private GameObject activatedVisual;
    [SerializeField] private GameObject deactivatedVisual;
    
    private void Awake()
    {
        interactionType = InteractionType.Toggle;
        objectType = ObjectType.Lever;
        
        // 初始化视觉状态
        UpdateVisualState();
    }
    
    protected override bool PerformInteraction(GameObject interactor)
    {
        // 切换激活状态
        isActivated = !isActivated;
        
        // 更新视觉状态
        UpdateVisualState();
        
        // 触发相应的行为
        if (isActivated)
        {
            Execute(objectType);
        }
        else
        {
            Cancel(objectType);
        }
        
        return true;
    }
    
    private void UpdateVisualState()
    {
        if (activatedVisual != null)
            activatedVisual.SetActive(isActivated);
        if (deactivatedVisual != null)
            deactivatedVisual.SetActive(!isActivated);
        
        // 更新交互提示
        interactPrompt = isActivated ? "按 E 键关闭" : "按 E 键打开";
    }
    
    #region IPropBehavior 接口实现
    
    public void Execute(ObjectType type)
    {
        // 执行激活逻辑
        Debug.Log($"机关已激活: {gameObject.name}");
        
        // 可以通过事件系统触发其他游戏逻辑
        GameEvents.TriggerMechanismActivated(gameObject);
    }
    
    public void Cancel(ObjectType type)
    {
        // 执行停用逻辑
        Debug.Log($"机关已停用: {gameObject.name}");
        
        // 可以通过事件系统触发其他游戏逻辑
        GameEvents.TriggerMechanismDeactivated(gameObject);
    }
    
    public void Cleanup()
    {
        // 清理资源
    }
    
    #endregion
}

## 6. 交互事件系统

```csharp
/// <summary>
/// 游戏事件类，用于处理交互相关的事件
/// </summary>
public static class GameEvents
{
    // 机关激活事件
    public static event Action<GameObject> OnMechanismActivated;
    
    // 机关停用事件
    public static event Action<GameObject> OnMechanismDeactivated;
    
    // 物品拾取事件
    public static event Action<ItemData, int> OnItemPickedUp;
    
    // 触发机关激活事件
    public static void TriggerMechanismActivated(GameObject mechanism)
    {
        OnMechanismActivated?.Invoke(mechanism);
    }
    
    // 触发机关停用事件
    public static void TriggerMechanismDeactivated(GameObject mechanism)
    {
        OnMechanismDeactivated?.Invoke(mechanism);
    }
    
    // 触发物品拾取事件
    public static void TriggerItemPickedUp(ItemData item, int quantity)
    {
        OnItemPickedUp?.Invoke(item, quantity);
    }
}
```

## 7. 性能优化策略

1. **交互区域优化**：使用更小的碰撞区域来标记交互点，而不是整个物体
2. **空间划分**：对大型场景使用空间划分技术（如四叉树）优化可交互物体的搜索
3. **状态缓存**：缓存可交互状态，避免每一帧都进行复杂的可交互性检查
4. **事件池**：对频繁触发的交互事件使用对象池
5. **延迟加载**：对远处的可交互物体使用延迟加载技术

## 8. 实现要点与注意事项

1. **标签与层**：为所有可交互物体添加"Interactable"标签，方便识别
2. **物理设置**：确保可交互物体的碰撞体设置正确，避免交互检测问题
3. **性能监控**：监控交互检测的性能，特别是在大型场景中
4. **可测试性**：为交互系统添加调试工具，方便测试不同交互情况
5. **安全性**：在交互执行前进行充分的有效性检查，防止空引用等错误

## 9. 扩展与兼容性考虑

1. **多种交互方式**：支持不同的交互方式（如长按、多次点击等）
2. **条件交互**：支持基于游戏状态、物品拥有情况等条件的交互
3. **多人交互**：考虑多玩家同时交互同一物体的情况
4. **跨场景交互**：支持在场景切换时保持交互状态
5. **网络同步**：为多人游戏预留网络同步接口
