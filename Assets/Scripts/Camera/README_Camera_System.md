# 摄像机跟随系统 - 方案一实现文档

## 概述
本摄像机跟随系统采用方案一配置，专为18×10瓦片显示区域设计，实现平滑的4瓦片水平死区和2瓦片垂直死区效果。

## 核心文件

### 1. PlayerFollowCamera.cs
**位置**: `Assets/Scripts/Camera/PlayerFollowCamera.cs`

**功能特点**:
- ✅ 方案一死区设置：4瓦片水平，2瓦片垂直
- ✅ 平滑跟随：使用SmoothDamp实现流畅移动
- ✅ 世界边界限制：防止摄像机超出游戏区域
- ✅ 缩放控制：支持鼠标滚轮缩放（4-8倍）
- ✅ 编辑器可视化：实时显示死区和边界

**关键参数**:
```csharp
// 方案一的标准配置
public float horizontalDeadZoneTiles = 4f;  // 22.2%屏幕宽度
public float verticalDeadZoneTiles = 2f;    // 20%屏幕高度
public float followSpeed = 8f;              // 跟随响应速度
public bool enableDeadZone = true;          // 启用死区功能
```

### 2. CameraFrustumCullingSystem.cs
**位置**: `Assets/Scripts/TileMap/CameraFrustumCullingSystem.cs`

**功能特点**:
- ✅ 视野范围计算和实时更新
- ✅ 瓦片碰撞体动态管理
- ✅ 性能优化：仅处理视野内对象
- ✅ 缓冲区域配置：防止频繁的剔除操作

### 3. CameraSystemController.cs
**位置**: `Assets/Scripts/Camera/CameraSystemController.cs`

**功能特点**:
- ✅ 自动系统集成和配置
- ✅ 统一参数管理
- ✅ 实时状态监控
- ✅ 运行时参数调整

## 死区机制详解

### 方案一配置（已实现）
```
水平死区：4瓦片 = 22.2%屏幕宽度
垂直死区：2瓦片 = 20%屏幕高度
总显示区域：18×10瓦片
死区比例：水平22.2%，垂直20%
```

### 工作原理
1. **中心区域**：玩家在死区内时，摄像机保持静止
2. **边界触发**：玩家超出死区边界时，摄像机开始跟随
3. **平滑移动**：使用SmoothDamp实现自然的跟随效果
4. **边界限制**：摄像机不能超出设定的世界边界

## 集成步骤

### 步骤1：添加摄像机组件
1. 在场景主摄像机对象上添加 `PlayerFollowCamera` 脚本
2. 将玩家对象拖拽到 Target 字段
3. 设置世界边界范围

### 步骤2：配置视野剔除
1. 在场景中创建空对象，添加 `CameraFrustumCullingSystem` 脚本
2. 关联主摄像机对象
3. 设置视野缓冲区大小

### 步骤3：系统集成（可选）
1. 创建空对象，添加 `CameraSystemController` 脚本
2. 自动配置或手动关联各个组件
3. 设置自动配置选项

## 参数配置建议

### 推荐设置
```csharp
// PlayerFollowCamera
horizontalDeadZoneTiles = 4f    // 水平死区：4瓦片
verticalDeadZoneTiles = 2f      // 垂直死区：2瓦片
followSpeed = 8f                // 跟随速度：适中
minZoom = 4f                    // 最小缩放：4倍
maxZoom = 8f                    // 最大缩放：8倍

// CameraFrustumCullingSystem
frustumBuffer = 5               // 视野缓冲区：5瓦片
enableDeadZone = true           // 启用死区
```

### 性能优化
- 视野缓冲区建议设置为3-8瓦片
- 跟随速度建议控制在5-12之间
- 缩放范围建议限制在合理区间
